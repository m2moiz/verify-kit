<objective>
Land the template's own self-test CI (DOC-04 / ROADMAP SC1): a GitHub Actions workflow that on every PR runs `copier copy` across the 5 add-on combinations and asserts `just verify` exits 0 in each scratch project. Land the sibling macOS workflow that re-runs the same matrix weekly. Land a local `act` validation script per ROADMAP SC5.

Purpose: The trust-anchor closure — verify-kit verifies itself. A regression in any template change is caught by the same `just verify` ground-truth command that consumers use. The matrix exercises 06-02 (auth)/06-03 (summarize defenses)/06-04 (echo hardening) end-to-end via the `+backend` and `+backend+llm` rows.

Output: Two workflow YAMLs + one validation script.
</objective>

<constraints>
- D-01: 5 rows on Linux per-PR + nightly macOS rerun on weekly cron.
- D-02: matrix entries exactly: `base`, `+backend`, `+llm`, `+backend+llm`, `+backend+llm+logfire+fastapi_mcp` (named base/backend/llm/backend-llm/full in the YAML per §5).
- D-03: wall-clock < 10 min per PR (ROADMAP SC5).
- D-04: weekly cron precedent `0 4 * * 0` (Sun 04:00 UTC).
- §5 Research: canonical YAML shape; `fail-fast: false`; `copier copy --trust --defaults --data-file <(echo '...')`; `jdx/mise-action@v2` for toolchain.
- §11 Research: `act` validation command — `act pull_request -W .github/workflows/template-selftest.yml -j selftest --matrix combo:base` (just `base` row for cheap validation).
- 06-02 contract: matrix rows with has_backend=true MUST either set `VERIFYKIT_AUTH_TOKEN=dev-token-for-tests` in the job env OR rely on the dev-mode fallback (ENV=dev + token unset → allowed). Pick one strategy and document it in the workflow's comment header.
- REVIEW-CHECKLIST §1: no cwd-relative paths in the workflow shell scripts. Use `$GITHUB_WORKSPACE`, `/tmp/scratch-<combo>`, etc. (absolute paths).
- REVIEW-CHECKLIST §5: any Jinja-like substitution must be on its own line (no merge risk in YAML). The `<(echo '...')` process substitution is bash-level inside a multi-line shell `run:` block — already on its own line.
- REVIEW-CHECKLIST §8: `just verify` inside the scratch project should NOT leak the outer-runner env into its subprocess calls. The matrix sets `working-directory: /tmp/scratch-<combo>` — uv/just resolve from there; no `VIRTUAL_ENV` leak in clean GHA runners (issue is more severe in local pytest contexts).
- act-local limitation (§11 Pitfalls): act runs Linux only — macOS workflow cannot be validated via act. The Linux `base` row IS validateable via act and is the canonical SC5 check.
- mise/Testcontainers limitation (§5 Pitfalls): macOS runner Testcontainers paths need skip-on-no-docker handling — verify Phase 4's existing skip pattern activates correctly. If a row hangs on macOS, that's a Phase 4 regression, not this plan's concern; surface as a beads issue.
</constraints>

    Create `.planning/scripts/act-validate-selftest.sh`:

    ```bash
    #!/usr/bin/env bash
    # ROADMAP SC5: confirm template-selftest.yml runs end-to-end via `act` locally.
    # Default: `base` combo only (cheapest, fastest sanity check, ~5 min).
    # --full: all 5 combos sequentially (SC5 literal — "full matrix in <10 min via act locally").
    #         Operator runs --full once before release; routine PR-local checks stay fast.
    set -euo pipefail
    ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../.." && pwd)"
    cd "$ROOT"

    FULL=0
    if [[ "${1:-}" == "--full" ]]; then FULL=1; fi

    if ! command -v act >/dev/null 2>&1; then
      echo "ERROR: act is not installed. Install: brew install act"
      exit 2
    fi

    echo "Listing workflows triggered by pull_request..."
    act -l pull_request

    if [[ "$FULL" -eq 1 ]]; then
      COMBOS=(base backend llm backend-llm full)
      echo
      echo "Running template-selftest.yml FULL matrix (${#COMBOS[@]} combos) under act..."
      START=$(date +%s)
      for combo in "${COMBOS[@]}"; do
        echo
        echo "=== combo: $combo ==="
        act pull_request \
          -W .github/workflows/template-selftest.yml \
          -j selftest \
          --matrix "combo:${combo}"
      done
      ELAPSED=$(( $(date +%s) - START ))
      echo
      echo "Full matrix elapsed: ${ELAPSED}s (SC5 budget: 600s for the 10-min target)"
      if [[ "$ELAPSED" -gt 600 ]]; then
        echo "WARNING: full matrix exceeded 10-minute SC5 budget"
      fi
    else
      echo
      echo "Running template-selftest.yml combo=base under act (default fast path)..."
      echo "Use --full to run all 5 combos for the SC5 release check."
      act pull_request \
        -W .github/workflows/template-selftest.yml \
        -j selftest \
        --matrix combo:base
    fi
    ```

  <action>
    `schemathesis.openapi.from_asgi("/openapi.json", app)` triggers a GET request via TestClient(app) WITHOUT entering lifespan. The `profile_middleware` at `app/main.py:76` reads `app.state.settings` which is lifespan-bound, so AttributeError.

    The minimal fix: wrap the schemathesis call in an `asyncio.run(...)` block that enters `LifespanManager(app)` first, like:

    ```python
    import asyncio
    from asgi_lifespan import LifespanManager

    async def _load_schema_with_lifespan(app):
        async with LifespanManager(app):
            return schemathesis.openapi.from_asgi("/openapi.json", app)

    schema = asyncio.run(_load_schema_with_lifespan(app))
    ```

    Alternative (simpler, possibly less correct): override `require_auth` globally on the imported `app` BEFORE the schemathesis call, AND patch `app.state.settings` with a dummy Settings instance so profile_middleware doesn't crash. This sidesteps lifespan but couples the harness check to the auth + profile middleware internals.

    Prefer the LifespanManager approach — it's the documented FastAPI pattern, requires no awareness of which middlewares read state, and is robust to future additions.

    Re-export the schema such that the rest of `backend_inprocess_fuzz.py:main()` continues to work unchanged. The schema lifecycle is short — it's only used to derive operations.

    Per Pattern 6: any new comment text uses evergreen wording, no planning provenance.
